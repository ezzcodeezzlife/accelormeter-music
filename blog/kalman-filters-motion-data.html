Using Kalman Filters for Smoother Motion Data

Published: March 2024

The Problem with Raw Sensor Data

When working with accelerometers, you'll notice that the raw data can be quite noisy. Even when the device is perfectly still, the readings fluctuate. This noise comes from various sources: electrical interference, mechanical vibrations, and the inherent limitations of MEMS sensors.

What is a Kalman Filter?

The Kalman filter is an algorithm that uses a series of measurements observed over time to estimate unknown variables more accurately than using a single measurement alone. It was developed by Rudolf Kalman in 1960 and has been widely used in navigation, robotics, and signal processing.

How It Works

The Kalman filter works in two main steps:

  1. Prediction: Based on the previous state, predict what the next measurement should be
  2. Update: Combine the prediction with the actual measurement, weighting each based on their uncertainty

Key Parameters

Our implementation uses two important parameters:

  • R (Measurement Noise Covariance): Represents how much we trust the measurement. Lower values mean we trust the sensor more.
  • Q (Process Noise Covariance): Represents how much we expect the value to change between measurements. Higher values allow for more responsiveness.

In Our Application

We use the kalmanjs library with R=0.01 and Q=3 to filter the raw Z-axis acceleration before calculating speed. This results in much smoother visualizations and more accurate speed calculations.

Conclusion

Kalman filtering is a powerful technique for dealing with noisy sensor data. While the mathematics can be complex, libraries like kalmanjs make it easy to integrate into web applications.
